# mac-timebackup Agent Rules

## 文档归档

- 根目录只保留入口级文档，例如 `README.md`、`AGENT.md`、许可证和工具链配置文件。
- 临时计划文档在实现完成后必须清理，不要长期留在根目录或 `docs/` 中。
- 计划文档包括但不限于 `plan.md`、`*-plan.md`、`TDD_PLAN_*.md`、issue 实施计划和迁移计划。
- 如果计划里的内容需要长期保留，先改写成稳定文档，再归类到 `docs/`。
- 架构决策放入 `docs/adr/`，使用递增编号和简短标题。
- 修复总结默认在修复完成后清理；只有改写成稳定操作说明或设计说明后，才放入 `docs/`。
- 领域语言和长期上下文放入 `docs/CONTEXT.md`；需要理解业务概念时读取它。
- 发布流程和 Homebrew tap 决策放入 `docs/release.md`；实现打包、CI 或发布相关改动前先读取它。

## 工作习惯

- 修改代码前先确认当前 git 状态，不要 stage 或回滚无关改动。
- 已实现的计划文档应在同一轮收尾中删除，避免后续 agent 误把旧计划当成当前事实。
- 文档调整要保持小而明确；不要为了归档顺手重写无关内容。
